Röviden és tömören

Az alkalmazás egy vállalatirányítási rendszer már elkészült részeit és azok kapcsolatait hivatott bemutatni. A Partners és Employees modulban lehet rögzíteni és lekérdezni a vállalat partnereinek és alkalmazottainak adatait. Ezek szorosan kapcsolódnak az A/P invoice / Szállítói számla részhez, melyek nélkül nem is kerülhet be számla a rendszerbe, emiatt egy-egy partnernél is megtekinthetőek a tőle befogadott számlák. A könyvelési rész fogja kezelni a számlákon végzett műveletek eredményét dátummal és számlaosztályokkal.

Technikai tartalom

Célom volt, hogy az alkalmazás kisebb, egymástól függetlenül telepíthető részekből épüljön fel. A webszerveren Docker keretrendszerben, 3 külön konténerben futó alkalmazás biztosítja a perzisztenciát, az üzleti logikát és a grafikus felhasználói felületet.

  • A futáshoz szükséges relációs adatbázis a MariaDB jelenlegi legfrissebb, 10.6-os Docker image-éből készült.
  • A backend réteget egy klasszikus Java Spring Boot alkalmazás valósítja meg. A Docker image az AdoptOpenJdk image-éből készült.
    • Fejlesztő és build eszköz: Maven
    • Adatbázis táblák létrehozása és verziókezelése: Flyway
    • Adatbázis műveletek végrehajtása, a Java objektumok és az adatbázis rekordok közötti megfeleltetés: Spring Data Jpa
    • Autentikáció és autorizáció: Spring Security
    • Http kommunikáció fejlesztése és tesztelése: Swagger
    • Fejlesztést segíti: Lombok, SonarLint, Git...
  • A grafikus felhasználói felület az Angular 12 keretrendszerben épül, és NGINX webkonténerben fut.

Jelenlegi modulok

PARTNERS

Itt lehet rögzíteni és lekérdezni a cég partnereinek adatait. A partnerek adatai szerepelnek a szállítói számlákon.

EMPLOYEES

Itt lehet rögzíteni és lekérdezni a cég alkalmazottainak adatait. Alkalmazottak tudnak szállítói számlákat rögzíteni, így kapcsolódnak a berögzített számlákhoz.

A/P INVOICES

Itt lehet rögzíteni - partnerrel és alkalmazottal - és lekérdezni a szállítói számlákat.
